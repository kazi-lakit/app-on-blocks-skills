# GitHub Workflows Reference

> **Note:** GitHub workflow files are NOT generated by this skill. CI/CD pipelines are managed separately by the infrastructure team. This document is **reference only** — it documents the workflow patterns used by the infrastructure team.

---

## Deployment Pipeline

```
GitHub Actions → Azure Container Registry (ACR) → Azure Kubernetes Service (AKS)
```

## Infrastructure-Managed Files

The infrastructure team manages:

| File | Purpose |
|------|---------|
| `.github/workflows/*.yml` | Deployment pipeline triggers |
| `.github/workflows/3_web.yml` | ACR build + AKS deploy |
| `.github/variables/vars.env` | Repository-level variables |
| `.github/actions/setvars/` | Environment variable loading action |

The skill's `references/dockerfile-template.md` documents the `--build-arg ci_build=${{ inputs.CI_BUILD }}` usage, but the workflows themselves are infra-managed.

## Dockerfile Variable

The `DOCKERFILE` environment variable in `.github/variables/vars.env` tells the ACR build which Dockerfile to use:

```
DOCKERFILE=Dockerfile
STORYBOOK_DOCKERFILE=storybook.Dockerfile
```

This allows the same workflow to build different container types from the same repo.
