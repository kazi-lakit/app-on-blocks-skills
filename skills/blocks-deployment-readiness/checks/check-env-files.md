# check-env-files

Verifies that a project has the required `.env` files with the necessary environment variables for deployment.

## When to Run

Part of the deployment-readiness check suite. Run alongside other check actions in parallel.

## How to Check

### Step 1: Look for Environment Files

Check for the existence of these files in the project root:

- `.env` — local default (may exist, optional)
- `.env.dev` — development environment
- `.env.stg` — staging environment
- `.env.prod` — production environment

Use glob patterns or file search to verify presence.

### Step 2: Verify Required Variables in Each Env File

Each env file (`.env.dev`, `.env.stg`, `.env.prod`) must contain:

**Required variables:**
- `VITE_API_BASE_URL` — API endpoint URL for the environment
- `VITE_X_BLOCKS_KEY` — Blocks project key (sensitive)
- `VITE_PROJECT_SLUG` — Project identifier
- `VITE_BLOCKS_OIDC_CLIENT_ID` — OIDC client ID for authentication

**Optional variables:**
- `VITE_CAPTCHA_SITE_KEY` — CAPTCHA site key
- `VITE_CAPTCHA_TYPE` — reCaptcha or hCaptcha
- `GENERATE_SOURCEMAP` — Whether to generate source maps
- `VITE_PRIMARY_COLOR` — Theme primary color
- `VITE_SECONDARY_COLOR` — Theme secondary color
- `VITE_BLOCKS_OIDC_REDIRECT_URI` — OIDC redirect URI

### Step 3: Check Variable Consistency

- All three env files should have the same variable keys (values differ per environment)
- `VITE_API_BASE_URL` should point to the correct environment endpoint (dev/stg/prod)
- The `.env` file (if present) should be the "effective" one — often a copy of the intended target environment for local dev

### Step 4: Security Check

- Ensure no real credentials are embedded in templates generated by AI — use placeholder comments like `# VITE_X_BLOCKS_KEY = <get from blocks CLI>`
- The env file templates should guide users to fill in values via CLI, not manually paste sensitive data

## Output Format

Return a structured report:

```
.env files check:
✅ .env.dev — Present (5 required vars found)
✅ .env.stg — Present (5 required vars found)
✅ .env.prod — Present (5 required vars found)
⚠️  VITE_BLOCKS_OIDC_REDIRECT_URI — Missing in .env.dev
⚠️  VITE_CAPTCHA_SITE_KEY — Optional, missing across all env files
```

## If Missing

If any env files are missing or have missing variables:

1. Offer to generate env file templates from `references/env-variables.md`
2. Remind the user that credentials should be set via CLI (`blocks login`, `blocks config set`)
3. Do NOT generate files with real credentials — use placeholder comments

## Reference

See `references/env-variables.md` for full variable documentation and `references/canonical-example.md` for the canonical example.

## Verification Checklist

After checking env files, verify your report includes:

- [ ] Each env file checked individually (.env.dev, .env.stg, .env.prod)
- [ ] Required variables verified: VITE_API_BASE_URL, VITE_X_BLOCKS_KEY, VITE_PROJECT_SLUG, VITE_BLOCKS_OIDC_CLIENT_ID
- [ ] Optional variables noted: VITE_CAPTCHA_SITE_KEY, VITE_CAPTCHA_TYPE, VITE_BLOCKS_OIDC_REDIRECT_URI, GENERATE_SOURCEMAP
- [ ] Placeholder values detected (<get from...>, <placeholder>) — not real credentials
- [ ] VITE_API_BASE_URL values match environment (dev/stg/prod endpoints)
- [ ] Structured output format used (✅ present, ⚠️ optional missing, ❌ required missing)
- [ ] If generating templates: placeholder comments used, no real credentials
- [ ] If generating templates: CLI guidance provided (blocks init --env)
